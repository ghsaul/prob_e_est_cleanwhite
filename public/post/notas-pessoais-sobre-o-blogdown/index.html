Notas pessoais sobre o Blogdown

Por Gabriel Holmer Saul em 17/03/2020

Índice

.

1.1 - Instalação do Blogdown e Hugo

install.packages('blogdown') # Instala o Blogdown
blogdown::install_hugo() # Instala o Hugo
blogdown::update_hugo() # Pode ser usado para atualizar o Hugo
# Talvez seja uma boa ideia rodar esse codigo logo depois de instalar o Hugo, só pra garantir

1.2 - Criando o site

new_site(dir="nome_do_diretorio_do_site")

O comando acima cria um site novo com especificações passadas nos argumentos dessa função. Para criar seu primeiro site, basta rodar o comando.

O diretorio que irá conter o site precisa não existir ou estar vazio. Se não for especificado o argumento dir, o comando tentará criar o site no diretorio atual, que muito provavelmente nao estara vazio. Por isso, use o argumento dir = "nome_da_pasta_do_site", dentro da função.

1.3 - RStudio Addins

O RStudio tem alguns Addins ali em cima para ajudar a programar o site em blogdown.

  • Server Site - Esse comando precisa ser rodado só uma vez, e faz o seu site recarregar toda vez que voce modifica-lo. Ele será mostrado na parte de View do RStudio, mas eu acho melhor visualizar no browser para trabalhar lado a lado com o código.

  • New Post - Cria um novo post, é bem intuitivo… (teste ele para ver o que acontece!)

  • Update Metadata -

  • Dá pra criar o projeto do site clicando em File -> New Project -> New Directory -> Website using blogdown. Caso isso seja feito, é importante que você vá em Tools -> Project Options -> Build Tools e desmarque as opções Preview site after building e Re-knit current preview when supporting files change.

1.4 - Opções globais

Esse capítulo não é muito útil, ele fala de algumas micro configurações que podem ajudar a facilitar a manutenção do site, mas são tão difíceis de aplicar que é melhor ter um pouquinho mais de trabalho mesmo.
